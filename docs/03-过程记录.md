# 03-过程记录

**目的**：对照本次协作中的真实决策、迭代与学习（可作为课程/答辩的过程说明）。

---

## 一、落地与协作方式

代码经 **GitHub → Vercel** 发布后，最深的体会是：**环境变量要对齐文档**——`DATABASE_URL`（池化）与 `DIRECT_URL`（迁移用直连）分离、`NEXT_PUBLIC_APP_URL` 与浏览器地址一致，否则迁移抢锁或登录跳转会莫名失败；养成 **先查 `DEPLOY.md` 再改生产** 的习惯。多次 **push `main` 触发部署**，把「改完即同步」当作默认节奏。

---

## 二、对话中做出或巩固的关键决策

| 方向 | 决定 | 简要理由 |
|------|------|----------|
| 数据与托管 | Next.js + Prisma + PostgreSQL，不用 SQLite | Serverless 托管需要可迁移的关系库 |
| 会话 | JWT + HttpOnly Cookie | 实现简单，适合无状态 API |
| 实时 | SSE（项目同步、私聊推送等） | 相对 WebSocket 在网关/Serverless 上更好落地 |
| AI | OpenRouter 统一调用 | 换模型方便，未配置则功能降级而非整站不可用 |
| 私信 | `DirectThread`/`DirectMessage` 存库；从项目侧栏改为 **跳转消息中心** | 聊天记录集中可查，与「任务讨论」职责分开 |
| 对象存储 | 交付物走 **Supabase Storage**，DB 存 `FileAsset`+`Attachment` 绑定任务 | 满足「文件属于任务」与项目资源汇总，避免大文件进数据库 |
| 权限 | 上传/删除与 **组织·项目角色**挂钩（如 `canEditTask`、`canDeleteDeliverable`） | 企业场景下读写分离 |

## 三、根据使用反馈做的调优（节选）

- **任务详情侧栏**：用 **flex 列布局 + `min-h-0` + `flex-1 overflow-y-auto`** 修复「滚不到底」；补充 **「保存」按钮** 与草稿 flush，缓解「自动保存感知弱」的问题。  
- **我的任务**：增加 **状态分栏 + 月历**，并把 **日历从项目工作台移出**，避免两处重复。  
- **项目内私聊**：改为 **联系人弹窗 + 跳转消息中心**（`?peer=` 深链打开会话），侧栏不再叠一层抽屉。  
- **协作入口**：项目管理增加 **一键复制项目 ID**，方便「加入项目」粘贴。  
- **交付物**：任务侧栏上传/拖拽、项目 **资源中心** 聚合筛选；未配 Storage 时明确提示而非静默失败。  
- **工程习惯**：侧栏改动画量、接口失败时在 **抽屉内展示错误**，避免用户看不到保存失败原因。

## 四、学习与反思

**巩固的**：React 组件拆分、REST + `fetch`、Zod 校验、Prisma 建模与迁移流程。**新接触或加深理解的**：Pooler 与直连在 **`migrate deploy`** 下的差异；Next **App Router** 里服务端组件与 **`"use client"`** 边界；**SSE** 客户端挂载位置（避免多处重复连接）；**Supabase Storage** 与服务端签名 URL；multipart 上传与 **`XMLHttpRequest` 上传进度**。**学法**：官方文档 + 构建/运行日志 + 对话里逐步收窄问题范围。**反思**：界面问题（滚动、保存感、入口不明显）往往来自真实使用，单靠自测不够；文档（如 `02-说明文档`）最好在功能稳定后再对齐修订。
